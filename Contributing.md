# Contributing Guidelines - AIC-SSI

Thank you for your interest in contributing to **AIC-SSI** (Self-Sovereign Identity Layer of Adaptive Intelligence Circle).

### Core Principles (Non-Negotiable)
- All contributions must respect **third path absolute**, **zero-donation**, and **ethical-from-kernel**.
- Identity must remain truly self-sovereign — no central authority, no government, no corporation control.
- Privacy and user autonomy are fundamental.

### How to Contribute
1. Fork the repository.
2. Create a feature branch.
3. Make your changes with clear documentation.
4. Open a Pull Request (PR) with detailed explanation.
5. All PRs will be reviewed by maintainers or the Founder.

**During the Founder’s military service (2027–2029):**  
Only maintenance, documentation, and bug fixes are accepted. No major architectural changes.

### What We Do NOT Accept
- Any attempt to add centralized control or backdoors.
- Contributions that weaken privacy or ethical principles.
- Commercial or dual-licensing proposals.

By contributing, you agree to this file, the Code of Conduct, and all policies in `/POLICIES`.
